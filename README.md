# AIOps 智能部署中台产品需求文档 (PRD)

## 1. 文档概览
### 1.1 文档目的
本文档旨在详细描述 AIOps 智能部署中台的功能需求、业务流程及技术架构。本版本重点集成 Ansible 自动化能力，并强化智能体在部署流程中的核心地位。

### 1.2 项目背景
传统的自动化运维工具（如 Ansible）虽然解决了批量执行问题，但在面对复杂部署方案的解析和动态决策时仍显不足。本项目通过将 Ansible 的执行力与 LLM 的理解力结合，实现“方案即部署”的智能化体验。

---

## 2. 产品定位与目标
### 2.1 产品定位
集成 Ansible 的智能 AIOps 部署中台。

### 2.2 核心目标
1. **方案自动化解析**：通过智能体解析上传的部署方案，自动匹配主机与资产。
2. **Ansible 深度集成**：内置主机清单管理、资产自动分发及 Playbook 批量执行。
3. **智能体驱动工作流**：智能体作为工作流的原子步骤，参与决策与执行。

---

## 3. 核心功能模块

### 3.1 基础设施概览 (Dashboard)
*   **核心指标**：活跃服务器、运行中智能体、部署成功率、系统负载。
*   **新建部署入口**：支持上传部署方案（PDF/Markdown/Text），触发智能解析流程。

### 3.2 智能体中心 (Agent Hub)
*   **模型与能力管理**：集成国内外主流模型（DeepSeek, GPT-4o, Claude 等），定义智能体的角色。
*   **工作流集成**：智能体可作为工作流中的一个节点，负责逻辑判断、日志分析或方案解析。

### 3.3 主机与资产管理 (Host & Asset Management)
*   **资产纳管**：支持按分组管理服务器（Ansible Inventory 风格）。
*   **状态监控**：实时查看主机连通性及基础资源状态。
*   **资产分发**：整合 Ansible 的 `copy` 和 `unarchive` 模块，实现部署资产（Playbooks、配置文件等）到目标主机的自动分发。
*   **批量操作**：支持将选定资产批量分发至指定主机组的特定目录下，并自动处理目录权限。
*   **引用机制**：在 Playbook 或脚本中通过引用 ID 调用资产。

### 3.5 部署工作流 (Deployment Workflows)
*   **Ansible 步骤**：支持在工作流中直接添加“执行 Playbook”或“批量命令”步骤。
*   **智能体步骤**：支持添加“AI 方案解析”或“AI 风险评估”步骤。

### 3.6 技能库与 MCP (Skill Library & MCP)
*   **原子化能力**：环境配置、部署实施、测试验证、安全合规。
*   **MCP 集成**：集成 Model Context Protocol 服务，使智能体具备操作数据库、文件系统及远程服务器的能力。

---

## 4. 核心业务流程：智能部署
1. **上传方案**：用户在“新建部署”中上传部署方案文件。
2. **AI 解析**：智能体解析方案，识别出：
    *   目标主机组。
    *   所需部署资产。
    *   执行步骤（Playbook 逻辑）。
3. **资产分发**：系统根据解析结果，通过 Ansible 将资产分发至目标主机。
4. **批量执行**：调用 Ansible 执行部署逻辑。
5. **验证反馈**：智能体分析执行日志，确认部署是否成功。

---

## 5. 技术架构
*   **执行引擎**：Ansible (Python)。
*   **AI 引擎**：Gemini / DeepSeek / Claude。
*   **前端**：React 19 + Tailwind CSS 4.0 + Motion。
