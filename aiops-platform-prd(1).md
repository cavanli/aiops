> 产品名称：AIOps部署中台

## 1. 产品概述

### 1.1 产品定位

面向企业运维团队的**开源AIOps自动化部署平台**，通过AI能力实现运维自动化，降低部署门槛，提升交付效率。

### 1.2 核心价值

- **自动化**：环境初始化 → 自动部署 → 测试验证全流程自动化
- **智能化**：AI赋能的工作流、Agent、RAG能力
- **开源化**：完全开源，企业可自主部署与二次开发

### 1.3 目标用户

- 企业运维工程师
- DevOps团队
- SRE工程师
- 平台工程师

### 1.4 产品范围

| 类别 | 说明 |
|------|------|
| 部署规模 | 百级服务器 |
| 部署模式 | 纯开源，不涉及SaaS |
| 核心流程 | 环境配置 → 部署实施 → 测试验证 |

## 3. 功能模块详细设计

### 3.1 模型广场

#### 3.1.1 功能概述

统一管理各类大模型，提供模型接入、切换、监控能力。

#### 3.1.2 功能点

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| **模型接入** | P0 | 支持OpenAI、Anthropic Claude、DeepSeek、通义、文心、智谱等 |
| **自定义模型** | P0 | 支持接入任意兼容OpenAI API的模型 |
| **模型分组** | P1 | 按场景/用途分组管理 |
| **密钥管理** | P0 | 加密存储模型API Key |
| **调用统计** | P1 | QPS、Token消耗、响应延迟、错误率 |
| **密钥脱敏** | P0 | 前端展示API Key或SSH私钥时必须掩码处理，审计日志严禁明文密钥 |
| **国产化加密** | P2 | 预留对接商用加密机或国密算法接口 |
| **模型测试** | P1 | 在线测试界面，支持流式输出 |

#### 3.1.3 数据模型

```go
type Model struct {
    ID          string    `json:"id"`
    Name        string    `json:"name"`
    Provider    string    `json:"provider"`       // openai/anthropic/deepseek/qwen/etc
    ModelType   string    `json:"model_type"`     // chat/embedding
    Endpoint    string    `json:"endpoint"`       // 自定义模型时使用
    APIKey      string    `json:"api_key"`        // 加密存储
    GroupID     string    `json:"group_id"`
    Status      string    `json:"status"`         // active/inactive
    CreatedAt   time.Time `json:"created_at"`
    UpdatedAt   time.Time `json:"updated_at"`
}
```

### 3.5 MCP中心

#### 3.5.1 功能概述

MCP (Model Context Protocol) 服务管理中心，支持一键启动和管理MCP服务。

#### 3.5.2 功能点

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| **MCP Server管理** | P0 | 一键启动/停止/重启MCP服务 |
| **npx引擎支持** | P0 | 支持`npx`方式快速启动MCP |
| **MCP市场** | P1 | 常用MCP Server模板（文件系统、数据库、Git等） |
| **健康检查** | P0 | MCP服务状态监控 |
| **日志查看** | P1 | 实时日志查看 |

#### 3.5.3 MCP配置示例

```yaml
mcp_servers:
  - name: filesystem
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/data"]
    env:
      NODE_ENV: production
    auto_start: true
    
  - name: postgres
    command: npx
    args: ["-y", "@modelcontextprotocol/server-postgres", "postgresql://user:pass@localhost/db"]
    auto_start: false
```

## 4. 部署流程设计

### 4.1 环境配置

| 功能点 | 说明 |
|--------|------|
| **主机注册** | IP、SSH密钥、标签、描述 |
| **主机分组** | 按业务/机房/用途分组 |
| **环境变量** | 全局/分组/主机级别变量 |
| **依赖预装** | Docker、K8s Client、Python等 |

### 4.2 部署实施

| 功能点 | 说明 |
|--------|------|
| **部署模板** | 支持Helm、Docker Compose、Shell脚本 |
| **部署策略** | 滚动部署、重新部署、蓝绿部署 |
| **批量部署** | 批量选择主机，一键部署 |
| **灰度发布** | 逐步灰度到更多主机 |
| **部署回调** | 支持Hook机制（部署前后） |
| **原子性策略** | 支持“全成功或全失败”模式。若其中一台主机部署失败，根据策略自动触发全量回滚 |
| **状态快照** | 执行部署前自动记录当前环境关键状态（如容器版本、配置Checksum），作为回滚基准 |

## 9. 风险与挑战

1. **工作流引擎复杂度**：可视化编排引擎实现难度较高
2. **Agent可靠性**：LLM Agent在生产环境的稳定性需要验证
3. **MCP生态**：MCP Server生态尚在早期，需要自行封装
4. **离线环境交付**：私有网无法访问外部npm/pip源，需实现MCP Server离线容器化封装，镜像预拉取机制
