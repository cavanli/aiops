# 2026年交付部智能体建设规划 

---

## 一、建设目标与演进路线

**核心目标**：构建覆盖交付全链路（Phase 1 资产发现 → Phase 2 环境预检 → Phase 3 拓扑同步 → Phase 4 智能部署 → Phase 5 智能运维 → Phase 6 智能可观测的自适应智能体体系，推动交付部门从"人工操作"向"机器自治"跨越，系统性提升交付效率与质量。

**三阶演进路线**：

```
阶段一 单体智能体            阶段二 协同智能体            阶段三 自适应智能体
────────────────────  →   ──────────────────── →    ──────────────────────
独立功能 Agent             多 Agent Workflow 编排       自主决策 · 最小人工干预
人工触发为主                MCP 工具链共享               全场景覆盖 · 自适应学习
```

**核心技术底座**：

| 技术域 | 选型 | 说明 |
|-------|------|------|
| **大语言模型** | 发布当月开源最佳（30B/70B/70B+MoE）->claude | 主力推理；备选复用现场已采购星火/讯飞模型 |
| **Agent 开发平台** | Dify（主）-> Openclaw+CC系列 | 可视化 Workflow + Agent + RAG 三合一 |
| **工具接入** | MCP 协议 + Skill+CLI 轻量封装（MCPorter） | 标准化新工具；Skill.md 封装可释放 70% 上下文空间 |
| **知识引擎** | Dify 内置 RAG + Hub-Spoke 离线分发 | 总部加工 → 增量包分发 → 各离线现场按需加载 |
| **上下文管理** | GCC（Git 风格记忆）+ 动态过滤 + 工具延迟加载 | 跨会话/跨智能体知识共享；长程任务表现提升 ~14% |
| **推理服务** | vLLM（生产）/ Ollama（本地开发） | 复用现场已有 GPU/NPU 资源 |

## 二、六层智能体能力矩阵

| 智能体 | 定位 | 核心能力 | 推理策略 |
|--------|------|---------|---------|
| **Phase 1 · 资产发现** | 交付起点 · 构建资产底座 | Nmap IP 段普查 / IPMI-Redfish 裸金属发现 / SNMP-LLDP 网络拓扑 / Categraf 对话式部署与热加载 | Function Calling |
| **Phase 2 · 环境预检** | 环境守门员 · 全生命周期管理 | OS 七阶段巡检自动修复（PreCheck-Bot）/ 资产上线即巡检 / 中间件层扩展 | ReAct |
| **Phase 3 · 拓扑同步** | 全局视野 · 变更影响评估 | 应用拓扑自动构建 / 跨层级影响分析 / 部署状态实时同步 / 变更报告生成 | Function Calling |
| **Phase 4 · 智能可观测** | 感知神经 · 多维遥测分析 | 日志异常模式识别 / ES 集群智能优化 / 指标-日志-链路多源关联 / 告警降噪聚合 | ReAct |
| **Phase 5 · 智能部署** | 核心交付执行引擎 | 标准化部署文档 → LLM 质量校验 → 智能拆分重构 → Ansible Playbook 自动执行（LLM→Playbook 准确率实测 98.86%） | FC + Plan-and-Execute |
| **Phase 6 · 智能运维** | 最近"自治"目标 · 故障闭环 | 自主故障检测（星相告警触发）/ 全维根因分析 / 可落地修复方案建议 / 主动可靠性优化 | ReAct + 自愈 Workflow |

**已有探索资产**（建设基础）：

| 资产 | 状态 | 归属层级 |
|-----|------|---------|
| PreCheck-Bot（IPTSE 智能体）| ✅ 可发版 | Phase 2 · 环境预检 核心引擎 |
| Ansible MCP Server | 📦 开发完成 | Phase 2 · 环境预检 / Phase 5 · 智能部署 执行引擎 |
| stellar-categraf + Playbook | 📦 开发完成 | Phase 1 · 资产发现 核心采集器 |
| 日志分析智能体 / ES 智能体 | 🚧 Beta | Phase 4 · 智能可观测 核心能力 |
| 告警治愈回调分析智能体 | 🚧 Beta | Phase 6 · 智能运维 核心能力 |
