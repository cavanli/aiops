工作流编辑器：MVP 实现方式

工作流模块是最复杂的部分，MVP 阶段选哪种方案？

A

真实可视化画布（ReactFlow）

引入 ReactFlow 库，支持拖拽节点、连线、缩放

优点
  • 符合 PRD 可视化编排要求
  • 用户体验好
缺点
  • 实现复杂度最高
  • 单独占一个 Plan
B

简化版：节点列表 + JSON 编辑器

用列表方式管理节点顺序，高级用户可直接编辑 JSON

优点
  • 实现简单，快速上线
  • 后端 API 已就绪
缺点
  • 不符合 PRD 可视化要求
  • 用户体验较差
C

MVP 先做列表，v1.1 升级画布

MVP 用简化列表快速交付，v1.1 迭代替换为 ReactFlow 画布

优点
  • MVP 快速上线
  • 数据结构已为画布设计
缺点
  • 需要两次开发工作流页
